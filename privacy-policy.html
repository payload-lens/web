Payload Lens Privacy Policy

Payload Lens is a Chrome DevTools extension for inspecting REST and GraphQL API traffic in the browser.

Data Handled

Payload Lens can display API request and response data that appears in Chrome DevTools, including URLs, methods, status codes, headers, request bodies, and response bodies.

Local Processing

Payload Lens processes captured traffic locally in the browser extension. It does not send captured API traffic to Payload Lens servers.

Redaction

Payload Lens redacts common sensitive values before display, copy, and optional persistence. Redacted data includes common auth headers, cookies, sensitive query parameters, and common JSON fields such as tokens, passwords, secrets, session IDs, and API keys.

Local Storage

If Preserve log is enabled, redacted log entries are stored locally using chrome.storage.local. Ignored endpoint preferences and theme settings are also stored locally. Users can clear saved log entries from the extension UI.

Replay

Replay runs from the inspected page context and uses the live in-memory request data available during the current DevTools session. Preserved log entries are redacted and are not intended to retain secrets for future replay.

Third Parties

Payload Lens does not sell captured data and does not share captured API traffic with third parties.

Chrome Web Store Limited Use

Payload Lens' use of information received from Chrome APIs adheres to the Chrome Web Store User Data Policy, including the Limited Use requirements.

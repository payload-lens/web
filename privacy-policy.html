Payload Lens Privacy Policy

Last updated: May 23, 2026

Payload Lens is a Chrome DevTools extension for inspecting REST and GraphQL API traffic in the browser. This policy explains what data the extension handles, how it is used, and how users can control it.

Single Purpose

Payload Lens exists to help developers inspect, group, replay, compare, and copy browser API requests and responses inside Chrome DevTools.

Data Handled

Payload Lens can display API request and response data that appears in Chrome DevTools, including URLs, methods, status codes, headers, request bodies, and response bodies.

This data may contain personal or sensitive information if the inspected website sends that information through its API traffic.

How Data Is Used

Captured API traffic is used only to provide Payload Lens features, including endpoint grouping, GraphQL operation grouping, formatted inspection, response diffing, request replay, local log preservation, ignored endpoint preferences, and copy helpers.

Local Processing and No Account

Payload Lens processes captured traffic locally in the browser extension. It does not require an account and does not send captured API traffic to Payload Lens servers.

Chrome Permissions

Payload Lens uses the storage permission to save local preferences and optional redacted preserved logs. It uses the clipboardWrite permission only when the user clicks a copy action, such as copying a payload, header list, fetch snippet, cURL command, or diff value.

Redaction

Payload Lens redacts common sensitive values before display, copy, and optional persistence. Redacted data includes common auth headers, cookies, sensitive query parameters, and common JSON fields such as tokens, passwords, secrets, session IDs, and API keys. Redaction is a safety measure, but users should still review copied output before sharing it.

Local Storage, Retention, and Deletion

If Preserve log is enabled, redacted log entries are stored locally using chrome.storage.local. Ignored endpoint preferences and theme settings are also stored locally. Users can clear saved log entries and ignored endpoint preferences from the extension UI. Uninstalling the extension also removes extension-local storage according to Chrome's extension storage behavior.

Replay

Replay runs from the inspected page context and uses the live in-memory request data available during the current DevTools session. Preserved log entries are redacted and are not intended to retain secrets for future replay.

Sharing and Sale of Data

Payload Lens does not sell captured data and does not share captured API traffic with third parties. The extension does not use captured data for advertising, tracking, profiling, or analytics.

Website and Feedback Links

The Payload Lens website is a static promotional site. It does not use product analytics or advertising cookies. The website loads fonts from Google Fonts. If users click external links such as Chrome Web Store, GitHub, or Feedback, those third-party services may process information according to their own policies.

Chrome Web Store Limited Use

Payload Lens' use of information received from Chrome and Google APIs adheres to the Chrome Web Store User Data Policy, including the Limited Use requirements.

Changes

If this policy changes, the updated version will be posted on this page with a new Last updated date.

Contact

Questions or privacy requests can be sent through the Payload Lens feedback form.
